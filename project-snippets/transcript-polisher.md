## Project Skills

- 전사본, 강의 대본, 자막, 회의록, 긴 산문을 문맥 흐름에 맞게 직접 읽고 다듬어야 할 때는 `<skills-root>/skills/transcript-polisher/SKILL.md`의 공유 스킬을 사용한다.

## Project Skill Overrides

- `transcript-polisher`를 사용할 때 코드 치환표, regex, 일괄 문자열 치환을 편집 엔진으로 쓰지 않는다.
- 스크립트는 파일 목록, 줄 번호, 문단 수, 검증 결과를 확인하는 보조 용도로만 사용한다.
- source root와 output root를 먼저 정하고, 병렬 출력이면 상대 경로를 그대로 보존한다.
- 별도 요약 요청이 없으면 `polish` 모드로 처리하고 원문 흐름과 정보량을 과도하게 줄이지 않는다.
- 반복되는 고유명사와 기술 용어는 `확정 용어`와 `불확실 용어`로 나눠 검토하되 자동 치환표처럼 쓰지 않는다.
- output을 재생성하거나 패치를 재적용할 때는 수정 대상이 source root가 아니라 output root인지 먼저 확인한다.
- 긴 텍스트는 문단, 구문, 강의 흐름 단위로 나누고, 사용자가 요청했거나 활성 runtime 정책이 자동 위임을 허용할 때만 단위마다 subagent를 배정한다.
- 강의 예시, 모델이 일부러 만든 어색한 출력 예시, 원문의 의도적 비문은 자동으로 매끈하게 고치지 말고 보존 여부를 판단한다.
- 긴 검토는 Claude Code `/goal` 방식처럼 완료 조건, 증거, 미충족 항목, 재검토 루프를 명시한다. 이는 자가 개선 스킬이 아니라 대화에 드러난 증거로 완료 여부를 감사하는 루프다.
