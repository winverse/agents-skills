project-workflow

Workflow suite의 setup 절반입니다. 새 프로젝트나 큰 initiative를 바로 구현으로 넘기지 않고, domain language, product reason, architecture boundary, design direction, PRD, issue backlog, workflow-state cache, spec handoff까지 준비시키는 초기 셋팅 orchestration입니다.

초기 셋팅 전용 workflow-state cache source-labeled primitive spec-workflow handoff no global install
agent skills/project-workflow/SKILL.md
human skills/project-workflow/skill.html
done setup gate가 끝나면 다음 단위는 spec-workflow로 넘깁니다.

사용 판단

raw idea와 project bootstrap은 사용하고, 반복 구현과 QA는 넘깁니다.

사용

새 서비스, 새 앱, 큰 initiative, cross-agent project setup, domain/architecture/design/PRD/issue backlog 준비.

결합

folder/env/db/infra 경계가 필요하면 project-structure, docs drift는 sync-docs, eval은 agent-eval-harness.

건너뜀

작은 버그 수정, 이미 준비된 issue 구현, TDD loop, review, browser QA, diagnose 요청.

확인

cloud write, destructive action, secret, MCP/API/file-write automation, untrusted content가 있으면 Agent Tool And Security Risk Gate부터 기록.

실행 흐름

먼저 프로젝트 언어와 이유를 고정하고, 구조와 PRD는 그 다음에 다룹니다.

기본 setup 흐름 위험 또는 구조 조건부 분기 핵심 gate handoff

입력과 출력

사용자가 준 아이디어를 durable artifact와 다음 workflow step으로 바꿉니다.

입력

  • raw idea, 새 서비스, 큰 initiative
  • 기존 README, docs, ADR, issue tracker
  • agent instruction surface: AGENTS.md, CLAUDE.md 등
  • UI, infra, MCP/API/file-write 여부

읽는 기준

  • root instruction과 folder-local instruction
  • references/upstream-dependency-map.md
  • references/project-workflow-playbook.md
  • project docs, design 기준, env/API/DB/deploy docs

출력

  • Runtime adapter와 active instruction surface
  • Primitive inventory with source package
  • workflow-state.md handoff cache
  • Project setup state와 artifact path
  • Next workflow step: remaining setup gate 또는 spec-workflow handoff

우선순위 차트

초기 셋팅에 직접 필요한 primitive만 고르고, 구현 loop는 의도적으로 제외합니다.

핵심성은 domain/product/handoff 순서로 높습니다.

local instructions + docs 필수
Matt Pocock / grill-me 핵심
GStack / office-hours 핵심
design.md + mock direction UI 조건
repo-local / project-structure 구조 조건
Agent Tool And Security Risk Gate 위험 조건
항상 먼저 봄 조건부 handoff 위험 gate

spec-workflow로 넘기기 전 준비도

domain term / boundary 필수
ADR or architecture note 필수
PRD or vertical issue 필수
design reference for UI UI 조건
tool/security decision tool 조건
validation command evidence 완료 조건
workflow-state.md cache handoff

빈 PRD나 막연한 "구현 시작"은 완료가 아닙니다. acceptance criteria와 validation evidence가 있어야 합니다.

파일 관계도

source instruction, reference, validator, agent metadata의 역할을 분리합니다.

agents/ OpenAI surface의 display name, short description, default prompt.
scripts/ repo 공통 package validator를 project-workflow에 연결하는 TypeScript entrypoint.

프로젝트 연결

global install이 아니라 target project instruction에 필요한 링크만 둡니다.

Use $project-workflow at skills/project-workflow/SKILL.md when starting a new project, raw product idea, major initiative setup, cross-agent project setup, domain/architecture/design/PRD/issue backlog preparation, or handoff into $spec-workflow.

프로젝트별 override는 target project의 AGENTS.md, CLAUDE.md, 또는 equivalent instruction file에 둡니다. shared skill에 특정 프로젝트 취향을 넣지 않습니다.

검증 게이트

workflow나 trigger가 바뀐 경우에는 skill package, Korean Markdown, HTML, eval까지 확인합니다.

로컬 검증 명령

  • node scripts/validate-skill.ts skills/project-workflow
  • node skills/project-workflow/scripts/validate-project-workflow.ts skills/project-workflow
  • node scripts/validate-korean-markdown.ts .
  • node scripts/validate-skill-html.ts .
  • node scripts/run-agent-evals.ts

완료 증거

  • source package와 exact primitive name이 inventory에 보임.
  • CONTEXT.md, ADR, PRD, issue, design, gate, workflow-state.md 중 필요한 artifact path가 명시됨.
  • UI나 tool/API 작업은 조건부 기준을 빠뜨리지 않음.
  • spec-workflow가 받을 spec 또는 remaining setup gate가 분명함.

금지와 허용

빠른 코딩보다 올바른 시작 조건을 우선합니다.

DO 해야 할 일

  • local instructions와 docs를 먼저 읽고 active instruction surface를 밝힙니다.
  • domain language를 stack choice보다 먼저 정리합니다.
  • 외부 primitive는 출처 패키지와 exact name을 함께 적습니다.
  • substantial UI는 design.md와 2-3 mock direction 선택을 먼저 둡니다.
  • 긴 bootstrap에는 /goal 또는 completion checklist를 measurable하게 씁니다.

DON'T 피할 일

  • raw idea를 곧바로 implementation TDD나 code changes로 넘기지 않습니다.
  • Matt Pocock/GStack 원문 prompt를 복사해 shared skill에 붙이지 않습니다.
  • project-structure를 domain/architecture 질문 전 raw discovery에 호출하지 않습니다.
  • tool, MCP, API, file write, network 작업의 Agent Tool And Security Risk Gate를 생략하지 않습니다.
  • commit, push, deploy, global install은 명시 요청 없이 연결하지 않습니다.
