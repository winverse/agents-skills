GitHub PR workflow

pull-request

커밋된 변경을 pull request로 제출하거나 기존 PR을 정리할 때 쓰는 스킬입니다. commit 생성은 atomic-committer, diff review는 code-review가 담당하고, 이 스킬은 branch/base/head, 한국어 title/body, reviewer, label, draft, issue link, gh CLI 실행 경계를 담당합니다.

사용 판단 매트릭스

사용

PR title/body 작성, `gh pr create`, draft PR, reviewer/label/milestone/project 지정, 기존 PR 상태 확인.

먼저 다른 스킬

커밋이 없으면 atomic-committer, 코드 리뷰가 목적이면 code-review를 먼저 사용합니다.

명시 요청 필요

merge, close, reopen, branch delete, force-push, auto-merge는 사용자가 직접 요청할 때만 수행합니다.

PR 생성 흐름

명령 선택표

새 PRgh pr create --base <base> --head <branch> --title ... --body-file ...
draftgh pr create --draft ...
reviewergh pr create --reviewer <user-or-team>
기존 PRgh pr view --json number,url,title,state,baseRefName,headRefName,isDraft

이 repo의 PR 규칙

언어PR title/body는 한국어로 작성하고 code identifier와 명령은 원문을 유지합니다.
스킬 문서skills/**/*.md는 한국어 우선이며 skill PR validation에 한국어 Markdown 검사를 포함합니다.
일관성SKILL.md, reference, HTML, snippet, history, eval, validator가 같은 behavior를 말해야 합니다.
goal 조건/goal은 Claude Code 기준입니다. 긴 PR 준비에서 template 준수, validation evidence, risk/rollback, PR URL 또는 명령 preview까지만 포함합니다.

파일 관계도

이 스킬은 SKILL.md + skill.html file pair를 기준으로 유지합니다.

프로젝트 연결 조각

- Use $pull-request at <skills-root>/skills/pull-request/SKILL.md when asked to prepare, create, update, draft, or manage a GitHub pull request.
- Use atomic-committer before pull-request when commits still need to be created.
- Run gh pr create only when the user explicitly asks to create a PR.
- In this skills repo, write Korean PR title/body and include Korean Markdown validation for skill changes.

금지와 허용

허용

PR body 초안, 명령 preview, draft PR, reviewer/label 지정은 요청 범위 안에서 수행합니다.

확인 필요

push가 필요한 branch, fork가 필요한 repo, project 권한이 필요한 옵션은 실행 전에 확인합니다.

금지

명시 요청 없이 merge, close, branch delete, force-push, auto-merge를 실행하지 않습니다.
