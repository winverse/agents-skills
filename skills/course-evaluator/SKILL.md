# 코스 평가자

이 스킬은 학생 답안 파일을 실제로 읽고, 루브릭과 사용자 채점 톤에 맞춰 학생별 평가 Markdown을 만든다. 핵심은 점수 산정과 교육적 코멘트를 분리하고, 각 학생의 실제 표현에 근거한 개별 피드백을 남기는 것이다.

상세한 채점 톤, 코멘트 구조, 반복 방지 기준이 필요하면 `references/scoring-feedback.md`를 읽는다.

## 핵심 계약

- CSV나 스프레드시트는 구조화 파서로 읽는다. 줄 단위 split으로 embedded newline 답안을 깨지 않는다.
- 기본 출력은 학생별 파일이다. 합쳐진 summary보다 `평가/학생명.md` 형태를 우선한다.
- 사용자 채점 정책을 먼저 고정한다: 만점 기준, 문항별 배점, 러프한 채점 여부, 합쇼체 여부, 코멘트 깊이.
- 점수는 관대하게 줄 수 있어도 코멘트는 얕아지면 안 된다.
- 코멘트는 각 학생이 실제로 쓴 표현을 반영한다. 같은 문장을 여러 학생에게 반복하지 않는다.
- 사용자가 `되게 잘`, `거의 모범 정답까지`, `놓친 부분으로`라고 말하면, 모범 답안에 가까운 보완 방향을 학생별 누락에 맞춰 써 준다.

## 실행 흐름

1. 입력 파일, 루브릭, 문항 수, 배점, 출력 폴더를 확인한다.
2. CSV는 구조화 파서로 파싱하고 학생 수와 문항 수를 검산한다.
3. 사용자 톤을 해석한다: `러프하게`, `합쇼체`, `거의 모범 답안`, `같은 스타일`, `학생별 파일`.
4. 학생별로 먼저 답안 요지를 읽고, 맞은 부분과 빠진 개념을 분리한다.
5. 점수를 산정한 뒤, 점수 이유와 고득점 보완 방향을 별도 문장으로 쓴다.
6. 학생별 Markdown 파일을 만든다.
7. 반복 코멘트, 누락 문항, 총점 합산, 파일 수를 검증한다.

## 출력 기준

학생별 파일은 다음 정보를 포함한다.

- 학생 이름
- 총점과 문항별 점수
- 문항별로 학생이 맞힌 핵심
- 문항별로 빠진 개념
- 고득점을 위한 보완 코멘트
- 사용자가 요청한 말투. 예: 합쇼체

## 반복 코멘트 방지 기준

- 학생 이름만 바꾼 stock phrase를 쓰지 않는다.
- 같은 루브릭 결론이라도 학생이 쓴 단어, 빠진 개념, 오해한 방향을 다르게 반영한다.
- 전체 학생을 처리한 뒤 비슷한 문장 시작, 같은 보완 문장, 같은 누락 설명이 반복되는지 다시 본다.
- 반복이 보이면 문항별 개념은 유지하되 학생 답안 근거를 다시 붙여 고친다.

## 검증 기준

```bash
node scripts/validate-skill.ts skills/course-evaluator
node skills/course-evaluator/scripts/validate-course-evaluator.ts skills/course-evaluator
node scripts/validate-korean-markdown.ts .
node scripts/validate-skill-html.ts .
node scripts/validate-skill-repo.ts .
node scripts/run-agent-evals.ts
```

실제 평가 작업에서는 다음도 확인한다.

- 입력 학생 수와 출력 Markdown 파일 수가 일치한다.
- embedded newline 답안이 보존됐다.
- 문항별 점수 합계와 총점이 일치한다.
- 반복 코멘트가 의심되는 문장을 다시 읽었다.
- 원본 답안이나 개인정보가 필요 이상으로 새 artifact에 복사되지 않았다.
