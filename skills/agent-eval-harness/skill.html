사용

행동 계약을 고정

스킬 라우팅, 안전 경계, 출력 형식, 파일 참조, 문서 동기화가 반복해서 깨질 수 있을 때.

함께 사용

project-workflow / spec-workflow

Workflow suite split, `workflow-state.md` handoff, spec loop, `/goal` 조건 품질을 regression case로 남길 때.

먼저 정함

live model/API 평가

예산, flake 정책, trace 저장, CI 차단 여부가 없으면 advisory 평가로만 둔다.

건너뜀

앱 기능 QA 대체

브라우저 QA, 제품 테스트, vendor trace platform 구축 자체가 목적이면 이 스킬의 범위가 아니다.

실행 흐름

완료 기준은 “하네스가 크다”가 아니라 case마다 성공 기준과 metric이 분명하고 deterministic gate가 실행되는 상태다.

입출력 스키마

사용자에게는 평가 범위, 추가한 case, 검증 명령과 결과를 짧게 보고한다.

오용 방지

허용
  • 먼저 success criteria를 쓰고 case schema에 반영한다.
  • 실제 실패를 가장 작은 regression prompt로 줄인다.
  • deterministic check를 먼저 blocking으로 둔다.
  • cross-agent portability 표면을 날짜와 함께 기록한다.
금지
  • 비밀값, 개인 경로, 원본 사용자 데이터를 fixture로 보존한다.
  • 불안정한 live API eval을 즉시 merge blocker로 둔다.
  • 한 모델의 말투에만 맞춘 brittle wording check를 만든다.
  • browser QA나 제품 기능 테스트를 이 하네스로 대체한다.
