금지와 허용

허용
  • 작은 case set으로 시작하고 baseline을 문서화한다.
  • real failure를 가장 작은 regression prompt로 승격한다.
  • 결정적 check를 먼저 blocking으로 둔다.
  • model/human grade는 보정 후 차단 조건으로 올린다.
금지
  • 실제 secret, 고객 데이터, production credential을 fixture에 넣는다.
  • raw trace, screenshot, exported report를 검토 없이 commit한다.
  • 불안정한 live API eval을 즉시 merge blocker로 둔다.
  • 한 모델의 말투에만 맞춘 brittle wording check를 만든다.
