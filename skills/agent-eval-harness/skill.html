agent-eval-harness

사용 판단 매트릭스

사용

초기 하네스

스킬 선택, prompt routing, guardrail, 출력 형식을 반복 검증하고 싶을 때.

함께 사용

스킬 운영

`agent-improvement-loop`나 `skill-update` 후 회귀 케이스를 남길 때.

주의

리뷰 품질, 디자인 판단, 리서치 종합은 advisory grade로 먼저 둔다.

건너뜀

플랫폼 구축

대시보드, 장기 trace 저장소, 유료 API 운영을 한 번에 만들려는 요청.

하네스 설정 흐름

입출력 스키마

입력

  • 반복 실패 prompt
  • 스킬/agent instruction 목록
  • 대상 agent surface와 날짜
  • 기존 test, validator, CI

처리

  • 성공 기준과 metric 정의
  • scope와 risk tier 선택
  • typical/edge/adversarial case 작성
  • deterministic check 우선

출력

  • `evals/agent/cases/*.json`
  • `evals/agent/artifacts/README.md`
  • `scripts/run-agent-evals.ts`
  • baseline과 artifact 정책

금지와 허용

허용
  • 작은 case set으로 시작한다.
  • 실패 prompt를 regression case로 승격한다.
  • 결정적 check를 먼저 blocking으로 둔다.
  • 판단형 grade는 human calibration 후 승격한다.
금지
  • 처음부터 대형 eval platform을 만든다.
  • 실제 secret이나 고객 데이터를 fixture에 넣는다.
  • raw trace와 screenshot을 그대로 commit한다.
  • 불안정한 live API eval을 바로 merge blocker로 둔다.
