design-review

Interline 취향과 화면별 규칙을 기준으로 정보 위계, 밀도, 타이포그래피, 안정적 레이아웃, 상태와 접근성을 검토하는 디자인 리뷰 스킬.

에이전트SKILL.md 사용자skill.html

사용 판단 매트릭스

사용

UI 품질 판단

hierarchy, density, spacing, typography, color, states, responsive order를 본다.

Interline

조용한 운영 UI

절제된 색, 얕은 border, 8px 이하 radius, stable dimensions를 선호한다.

함께 사용

브라우저 증거

실제 렌더링, console, network 문제는 browser-qa와 함께 확인한다.

건너뜀

구현 리뷰

코드 구조, 테스트, 보안은 code-review로 넘긴다.

리뷰 흐름

입출력 스키마

입력

  • screenshot/browser view
  • surface와 사용자 task
  • design-system context

리뷰 축

  • hierarchy/density
  • typography/color
  • states/responsive
  • Interline screen rules

출력

  • severity design findings
  • visual evidence
  • browser-qa follow-up

Interline 화면별 체크

Library / Book Detail

  • Book Series 우선
  • Book Cover 고정 비율
  • Section List와 cover 분리

Section Study

  • desktop: Sentence List | Study Panel | Study Settings
  • mobile: Study Panel 먼저
  • Book Translation/Vocabulary Note는 disclosure

Controls / Motion

  • Chunking Level은 실제 select
  • Coolicons family 일관성
  • Motion for React는 짧고 안정적으로

금지와 허용

허용
  • 정보 위계와 반복 작업 효율을 먼저 본다.
  • Interline 기준과 화면별 규칙을 적용한다.
  • 렌더링 의존 판단은 browser-qa 증거를 요구한다.
금지
  • 운영 UI를 landing page처럼 만들라고 하지 않는다.
  • decorative gradients/orbs/bokeh를 권하지 않는다.
  • 브라우저 동작을 증거 없이 단정하지 않는다.
