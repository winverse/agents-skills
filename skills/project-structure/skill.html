Stack principle

기본값은 빠르게 구조를 정하기 위한 출발점이다. 프로젝트 종류가 정해지면 관련 stack만 묻고, 나머지는 기본 정책으로 둔다. GraphQL codegen, Drizzle migration, Tauri build는 env helper를 반드시 거친다.

Folder Structure Trees

처음 설치했을 때 바로 비교할 수 있도록 frontend와 backend 기본 구조를 tree 형태로 보여준다. Route는 얇게, domain과 provider 경계는 별도로 보이게 한다.

Frontend web tree Next.js + urql 
apps/web/                           # 배포되는 Next.js 앱
├── env/                            # 앱별 env 파일 위치
├── codegen.ts                      # GraphQL 타입 생성 설정
├── scripts/                        # 컴포넌트 생성 등 자동화
└── src/                            # 애플리케이션 source root
    ├── app/                        # route, layout, error만 얇게 담당
    ├── config/                     # 앱 env 검증과 노출 경계
    │   └── env.ts                 # public/server env 분리
    ├── features/                   # 도메인 단위 기능 묶음
    │   └── posts/                  # posts 도메인
    │       ├── components/        # posts 전용 화면 조각
    │       ├── graphql/           # posts query/mutation 문서
    │       ├── hooks/             # posts 전용 client 로직
    │       └── prefetch/          # server prefetch 함수
    ├── graphql/               # urql client와 GraphQL autogen
    │   ├── client.ts             # urql client 설정
    │   └── autogen.ts         # web GraphQL autogen entrypoint
    ├── providers/                 # 전역 provider 조합
    └── components/                # 도메인 없는 공용 UI만
Backend API tree NestJS + Fastify + GraphQL 
apps/api/                           # 배포되는 NestJS API 앱
├── env/                            # API 전용 env 파일 위치
├── codegen.ts                      # GraphQL 타입 생성 설정
├── src/                            # NestJS source root
│   ├── main.ts                    # Fastify adapter 부트스트랩
│   ├── app.module.ts              # Nest root module
│   ├── config/                    # Zod env를 Nest 서비스로 제공
│   │   ├── env.ts                # app-local env schema
│   │   └── config.service.ts     # 검증된 env 주입
│   ├── common/                    # 공통 Nest 정책과 확장점
│   │   ├── guards/               # 인증/인가 guard
│   │   ├── filters/              # 예외 변환
│   │   ├── interceptors/         # request logging 등 횡단 관심사
│   │   └── pipes/                # 입력 검증/변환
│   ├── graphql/                   # GraphQL module과 autogen
│   │   ├── context.ts           # request context 구성
│   │   ├── graphql.module.ts    # GraphQL bootstrap
│   │   └── autogen.ts       # API GraphQL autogen entrypoint
│   ├── modules/                   # 도메인 module 모음
│   │   ├── auth/                # auth 도메인
│   │   │   ├── auth.module.ts   # auth Nest module
│   │   │   ├── auth.resolver.ts # GraphQL 요청 진입점
│   │   │   └── auth.service.ts  # auth 비즈니스 로직
│   │   └── users/               # users 도메인
│   └── providers/                 # 외부 서비스 adapter
│       ├── jwt/                   # JWT 발급/검증
│       ├── logger/                # log level, formatter, request logger
│       └── cache/                 # cache abstraction
│           └── redis/             # Redis client와 cache/session 연결
└── test/                           # API e2e 테스트
Full-stack monorepo package boundary Turborepo + Bun 
repo-root/                          # Bun + Turborepo workspace
├── apps/                           # 실행/배포되는 앱 모음
│   ├── web/                       # Next.js frontend 앱
│   └── api/                       # NestJS GraphQL backend 앱
├── packages/                       # 앱들이 공유하는 source
│   ├── config/                    # env helper만 제공, schema 금지
│   ├── db/                        # Drizzle schema, client, migration
│   ├── graphql/                  # 공유 schema 또는 generated 타입
│   ├── ui/                        # 앱 중립 UI primitive
│   ├── tsconfig/                  # 공유 TypeScript 설정
│   └── eslint-config/             # 공유 lint 설정
├── turbo.json                     # task pipeline
├── package.json                    # workspace scripts
└── bun.lock                        # dependency lockfile
folder or source boundary generated or GraphQL artifact tooling/config package

Env Contract

App-local schema apps/web/src/config/env.ts와 apps/api/src/config/env.ts가 각자 Zod schema를 가진다.
Helper-only package packages/config는 env mode, dotenv loading, Zod helpers만 제공한다. app 변수 정의는 하지 않는다.
No shared env reads 공유 패키지는 process.env를 직접 읽지 않는다. 필요한 값은 app에서 주입한다.
Frontend split browser 공개 값과 server-only 값을 분리한다. 공개 값만 framework public prefix를 사용한다.
Tooling uses helpers codegen, migration, Tauri build script가 각자 env를 다시 파싱하지 않고 공용 helper를 호출한다.
Per-app env files env/.env.development, .env.test, .env.production을 app마다 둔다.

Do / Don't

Do

  • 먼저 숫자로 프로젝트 종류를 확정한다.
  • route는 얇게, domain은 features에 둔다.
  • NestJS는 modules/common/providers 경계를 유지한다.
  • 중복 folder 역할을 최소화하고 app/domain/shared 경계를 명확히 한다.
  • 최종 tree가 선택한 folder 규칙을 지켰는지 검증한다.
  • API logger와 cache/Redis는 provider 경계 뒤에 둔다.
  • env helper를 codegen, migration, build에도 재사용한다.

Don't

  • 기본값으로 full-stack single repo를 제안하지 않는다.
  • shared package에서 process.env를 읽지 않는다.
  • GraphQL 기본 프로젝트에 REST controller를 기본 생성하지 않는다.
  • 기존 repo를 조사하지 않고 대량 이동하지 않는다.
