사용 판단 매트릭스

사용

새 구조 선택

frontend, backend, full-stack monorepo, desktop app, infra-aware repo의 folder tree와 boundary가 필요할 때.

질문

종류가 불명확

1 Frontend web, 2 Backend API, 3 Full-stack monorepo, 4 Desktop app처럼 숫자 메뉴로 좁힌다.

함께 사용

workflow 이후

raw idea 단계가 아니라 domain language와 architecture 질문이 생긴 뒤 workflow에서 구조 handoff로 호출한다.

중단

기존 코드 무시

repo 편집 시 기존 folder와 user code를 읽지 않고 tree를 덮어쓰면 안 된다.

질문 흐름

순서가 있는 handoff 완료 기준: tree, boundary, 검증 결과가 함께 있음

기본 스택

MongoDB 선택 시 Atlas와 repository boundary를, Supabase 선택 시 managed PostgreSQL과 server-only service-role boundary를 적용한다.

DB 옵션

PostgreSQL / Supabase Postgres

packages/db/drizzle, packages/db/src/postgres, packages/db/scripts/psql.ts를 둔다. Supabase service-role 값은 SUPABASE_SERVICE_ROLE_KEY secret name으로만 문서화한다.

MongoDB Atlas

packages/db/src/mongo가 client, collections, indexes를 소유한다. MONGODB_URI와 Atlas network assumptions를 서버 전용으로 둔다.

shell helper

PostgreSQL/Supabase는 psql, MongoDB는 mongosh를 쓴다. MongoDB에는 Drizzle migration을 만들지 않는다.

폴더 AGENTS.md

purpose

각 boundary folder의 목적과 소유 범위를 한 줄로 설명한다. 예: apps/api/AGENTS.md는 API module, provider, test boundary를 설명한다.

local map

하위 목차를 짧게 둔다. src/modules, src/providers, test처럼 agent가 먼저 볼 경로만 나열한다.

do-not-change

generated output, vendor, build artifact, live secret, cross-boundary owner를 이 folder에서 직접 바꾸지 않도록 적는다.

related skills

$project-structure, $sync-docs, $browser-qa처럼 이 folder에서 자주 쓰는 skill을 링크한다.

validation

이 folder에서 변경 후 실행할 typecheck, lint, test, codegen, migration, infra preview 명령을 적는다.

root 복사 금지

Root AGENTS.md 전체를 복제하지 않는다. folder-local 파일은 목차형 index로 유지한다.

env 계약

앱별 schema

모든 app은 <app>/env/*와 <app>/src/config/env.ts Zod schema를 가진다.

helper 전용 package

packages/config는 helper만 제공한다. shared package는 process.env를 직접 읽지 않는다.

도구도 동일 경로

GraphQL codegen, Drizzle migration, psql/mongosh shell, Tauri build, Infrastructure deploy script가 같은 env helper layer를 통과한다.

GraphQL codegen 계약

앱 소유 config

GraphQL app은 app root에 codegen.ts를 둔다.

operation 위치

frontend와 desktop query/mutation 문서는 src/features/<domain>/graphql 근처에 둔다.

autogen entrypoint

web, API, desktop 모두 <app>/src/graphql/autogen.ts를 import entrypoint로 쓴다.

인프라 정책

Pulumi AWS

infra/pulumi가 network, IAM, AWS ECR, ECS Fargate 또는 EC2 Docker host, load balancer, logs, runtime wiring을 소유한다.

Docker와 registry

<app>/Dockerfile과 .dockerignore는 app 옆에 둔다. CI는 Docker image를 immutable tag로 ECR에 push한다.

secret과 entrypoint

live secret은 Pulumi stack file에 넣지 않는다. required secret names, public entrypoint, Route 53, ACM, post-deploy smoke check를 문서화한다.

구조 검증

완료 전 확인
  • 선택한 project kind에 없는 app type이 tree에 끼지 않았는지 확인한다.
  • 최종 tree가 선택한 folder 규칙을 지켰는지 검증한다.
  • env와 GraphQL codegen path가 모든 app에서 같은 상대 위치인지 확인한다.
  • Drizzle, MongoDB, Supabase, Redis owner가 중복되지 않는지 확인한다.
  • infra-aware tree라면 public entrypoint, secret names, smoke check, runtime path가 보이는지 확인한다.

금지와 허용

해야 함
  • 숫자 메뉴는 tree를 바꾸는 선택에만 쓴다.
  • directory tree를 먼저 보여주고 app, feature, provider, shared package, generated files, scripts 경계를 설명한다.
  • 기존 repo를 편집할 때는 먼저 folder와 scripts를 읽고 user code를 보존한다.
하지 않음
  • full-stack single-repo layout을 기본으로 제안하지 않는다.
  • infra 요청이 없는데 Pulumi, Docker, AWS folder를 넣지 않는다.
  • local Compose를 production runtime처럼 설명하지 않는다.
  • folder-local AGENTS.md에 root instruction 전체를 복사하지 않는다.
