browser-qa

브라우저에서 실제 렌더링, 의도한 visible content, console, network, accessibility snapshot, viewport, text overflow를 확인하는 QA 스킬.

에이전트SKILL.md 사용자skill.html

사용 판단 매트릭스

사용

런타임 + 의도 검증

console, network, screenshot, accessibility snapshot, viewport, visible content 확인이 필요할 때.

함께 사용

디자인 리뷰

취향/위계 판단은 design-review와 분리하고, 실제 렌더 증거만 제공한다.

확인 필요

의도 불명확

URL, route, viewport, dev server, 기대 visible section이 불명확하면 먼저 좁힌다.

건너뜀

민감/파괴 작업

credential 입력, 결제, production destructive action은 하지 않는다.

검증 흐름

입출력 스키마

입력

  • URL 또는 HTML 파일
  • route와 viewport
  • 검증할 interaction

의도와 증거

  • visible headings/sections
  • console/network
  • accessibility snapshot
  • screenshot/viewport

출력

  • severity별 findings
  • 재현 단계
  • passed checks

파일 관계 지도

skills/browser-qa/ ├── SKILL.md ├── skill.html ├── agents/openai.yaml ├── references/browser-checklist.md └── scripts/validate-browser-qa.ts

프로젝트 연결 조각

project-snippets/browser-qa.md가 외부 프로젝트 연결 문구를 제공한다.

품질 게이트

HTTP 200 OK는 도달성 확인일 뿐이다. 완료 판단은 visible content가 의도와 맞는지까지 확인한 뒤 내린다.

node scripts/validate-skill.ts skills/browser-qa
node skills/browser-qa/scripts/validate-browser-qa.ts skills/browser-qa
node scripts/validate-skill-html.ts .
node scripts/validate-skill-repo.ts .

금지와 허용

허용
  • URL, viewport, repro step을 남긴다.
  • console과 network evidence를 확인한다.
  • visible content와 의도 검증 결과를 함께 남긴다.
  • 정적 HTML은 local server로 확인한다.
금지
  • 민감 정보를 입력하지 않는다.
  • 스크린샷만으로 결론내지 않는다.
  • 디자인 취향을 QA failure로 쓰지 않는다.
