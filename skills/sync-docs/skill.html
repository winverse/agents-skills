sync-docs

현재 repo 문서를 서로 대조해 오래된 설명, 누락된 연결, 충돌하는 규칙, 불명확한 source of truth를 정리합니다. 확실한 문서 mismatch는 고치고, 규칙 변경이 될 수 있는 충돌은 수정 전에 묻습니다.

skills/sync-docs/ SKILL.md - 에이전트 실행 규칙 skill.html - 사람이 읽는 시각 가이드

사용 판단 매트릭스

문서 동기화 요청을 받았을 때 수정, 비교, 조합, 질문을 빠르게 가릅니다.

사용

README, AGENTS, docs, history, snippets, skill guide, validator 설명이 서로 맞는지 검토하거나 최신화할 때.

확장 비교

folder-local AGENTS.md, hook 설명, lifecycle state, skill 목록, 경로, 명령이 여러 파일에 반복될 때.

질문

두 문서가 서로 다른 선호를 말하고 현재 파일, 코드, history만으로 의도를 판정할 수 없을 때.

조합 또는 중단

외부 최신 사실은 web-research, 코드 리뷰는 code-review와 조합합니다. 문서 밖 정책 변경은 단정하지 않습니다.

실행 흐름

작은 inventory에서 시작해 claim 단위로 근거를 모으고, 확신도에 따라 수정하거나 질문합니다.

우선순위 차트

문서 신뢰도와 실행 가능성을 깨는 항목부터 처리합니다.

입출력과 검증 스키마

입력은 문서와 로컬 근거, 출력은 동기화 diff와 짧은 보고입니다.

입력

  • 문서 최신화 또는 reconcile 요청
  • README.md, AGENTS.md, docs/
  • folder-local AGENTS.md instruction indexes, local map, validation commands
  • target project skill setup, selected SKILL.md links, neighboring skill.html files
  • project-snippets/, history/
  • skills/*/SKILL.md, guides, validators, hooks
→

출력

  • 확실한 stale 문서 수정
  • claim별 source of truth 판단
  • 수정 전 확인이 필요한 질문
  • 검증 명령과 pass/fail
  • 남은 불확실성과 의도적으로 둔 항목

프로젝트 연결

프로젝트 instruction 파일에는 필요한 경우만 짧게 연결합니다.

## Project Skills

- Use $sync-docs at <skills-root>/skills/sync-docs/SKILL.md when asked to review, refresh, reconcile, or update documentation by comparing existing docs, snippets, history, skills, root or folder-local AGENTS.md indexes, and repo instruction files.

## Project-Specific Overrides

- Treat the current repo files as the first evidence source.
- When folder structure changes, refresh the nearest folder-local AGENTS.md table-of-contents index and its local map, do-not-change boundaries, related skills, and validation commands.
- Ask before changing a documentation rule when the source of truth is unclear.
- Keep history ledgers durable; do not turn them into scratch review notes.
- Run available documentation, repo, and skill validators after edits.

허용

  • 문서끼리 먼저 비교하고 claim ledger(주장 기록)로 근거를 나눕니다.
  • stale 경로, 이름, validator 명령, 누락된 cross-reference는 작게 고칩니다.
  • local-only review note와 durable history 규칙은 보존합니다.
  • 마지막 보고는 Synchronized, Questions, Validation 중심으로 짧게 씁니다.

금지

  • 애매한 선호 충돌을 source of truth인 것처럼 임의 결정하지 않습니다.
  • history/skills.md를 임시 audit 메모처럼 재작성하지 않습니다.
  • 외부 최신 사실이 필요한 일을 로컬 문서 비교만으로 단정하지 않습니다.
  • 문서 sync 요청을 계기로 unrelated code, README, snippets, docs를 넓게 고치지 않습니다.
