sync-docs

현재 repo 문서끼리 대조해서 오래된 설명, 누락된 연결, 충돌하는 규칙을 찾고 확실한 것은 고치며 애매한 것은 사용자에게 묻는 문서 동기화 스킬

skills/sync-docs/ SKILL.md - 에이전트 흐름 skill.html - human guide

사용 판단 매트릭스

문서 변경 요청을 받았을 때 바로 수정할지, 비교할지, 질문할지 판단합니다.

사용

README, AGENTS, docs, snippets, history, skill guide가 서로 맞는지 검토하거나 최신화할 때.

비교

스킬 목록, 경로, validator 명령, hook 설명, lifecycle state가 여러 문서에 반복될 때.

확인 필요

두 문서가 서로 다른 선호를 말하고 현재 repo만으로 source of truth를 정할 수 없을 때.

건너뜀

문서가 아니라 코드 설계 자체를 정해야 하거나, 외부 최신 사실 조사가 핵심이면 다른 스킬과 조합합니다.

실행 흐름

먼저 현황을 모으고, claim 단위로 source of truth를 정한 뒤 확실한 mismatch만 고칩니다.

필수 흐름 질문 분기 확실한 수정

충돌 우선순위 차트

위에 있을수록 먼저 고칩니다.

입출력과 검증 스키마

입력은 문서 묶음, 출력은 수정과 질문, 검증 결과입니다.

입력

  • 문서 최신화 요청
  • README, AGENTS, docs
  • project-snippets
  • skills/*/SKILL.md와 guides
  • history, validators, hooks
→

출력

  • 확실한 stale 문서 수정
  • 충돌 claim과 source of truth 정리
  • 사용자에게 물어볼 질문
  • 검증 명령과 결과
  • 남은 risk 요약

파일 관계 지도

스킬 본문은 짧게 두고, 반복 audit 기준은 reference와 validator에 둡니다.

SKILL.mdtrigger, 실행 흐름, 질문 규칙, 출력 형태 doc-audit-checklistinventory, claim ledger, 충돌 분류 검증기정적 검사 project-snippets/sync-docs.md프로젝트 연결 블록 README.md공개 스킬 목록 기준 history/skills.md영구 lifecycle 기록

프로젝트 연결

프로젝트 instruction 파일에 필요한 경우만 붙입니다.

## Project Skills

- Use $sync-docs at <skills-root>/skills/sync-docs/SKILL.md when asked to review, refresh, reconcile, or update documentation by comparing existing docs, snippets, history, skills, and repo instruction files.

## Project-Specific Overrides

- Treat the current repo files as the first evidence source.
- Ask before changing a documentation rule when the source of truth is unclear.
- Run available documentation, repo, and skill validators after edits.

허용

  • 문서끼리 먼저 비교하고 claim ledger로 판단합니다.
  • 확실한 stale 경로, 명령, 스킬 목록은 바로 고칩니다.
  • 검증 명령과 남은 질문을 짧게 보고합니다.

금지

  • 애매한 선호 충돌을 임의로 결정하지 않습니다.
  • history ledger를 임시 검사 메모처럼 바꾸지 않습니다.
  • 외부 최신 사실이 필요한 일을 로컬 문서 비교만으로 단정하지 않습니다.
