terminal-session-automation

cmux, Warp, generic terminal에서 Codex hook, tab title, status, workflow note를 프로젝트 단위로 연결해 질문 원문을 안전하게 세션 표면에 고정하는 자동화 스킬.

사용 판단 매트릭스

사용

host별 세션 자동화

cmux tab/status, Warp title, generic shell title, workflow note를 만들거나 고칠 때.

실행까지

prompt pinning

사용자가 enable, install, test를 요청하면 recipe만 주지 않고 project-local hook과 dry-run을 실제로 갱신한다.

먼저 질문

강한 권한

global hook, socket auth, 다른 pane 입력, tab/workspace 종료, history 삭제, Warp/Oz/shell 설정은 승인 후 진행한다.

건너뜀

표면 없음

cmux env, Warp title output, `/dev/tty`가 없으면 조용히 성공 종료하고 Codex 흐름을 막지 않는다.

자동화 흐름

기본 구현 경로 hook 지연 방지 규칙

prompt pinning 입출력

입력

  • Codex `UserPromptSubmit` JSON stdin
  • 제출된 원문 prompt
  • `CMUX_SURFACE_ID`, `CMUX_WORKSPACE_ID`, `TERM_PROGRAM`

처리

  • 알려진 payload field에서 prompt 추출
  • 한 줄 whitespace normalization만 적용
  • cmux surface, Warp `/dev/tty`, generic no-op 순서로 처리

출력

  • `cmux rename-tab --surface ... "<original prompt>"`
  • `cmux set-status current-question "<original prompt>"`
  • Warp에서는 OSC title sequence만 출력
  • 필요 시 Codex `/hooks` 승인 안내

기본값은 원문 그대로 보존이다. 앞말 제거, 요약, 규칙 변환, 기본 절단은 하지 않는다.

제목 전략

우선순위 차트

`cmux identify`는 fallback이다. Warp와 generic terminal은 `/dev/tty`가 쓸 수 있을 때만 OSC title을 보낸다.

파일 관계 지도

프로젝트 연결

instruction 링크

프로젝트의 `AGENTS.md`, `CLAUDE.md`, 또는 동등한 instruction 파일에서 repo-local `SKILL.md`를 연결한다. 전역 설치나 symlink는 명시 요청이 있을 때만 한다.

Use $terminal-session-automation at <skills-root>/skills/terminal-session-automation/SKILL.md
when automating cmux, Warp, or generic terminal session titles,
status, hooks, workflow notes, or prompt pinning.

실제 hook 작업에서는 dry-run payload, host별 title/status 변경, foreground wrapper latency를 따로 확인한다.

오용 방지

허용 금지 승인 필요
project-local hook script와 config를 백업 후 갱신 원문 prompt를 기본으로 요약, 앞말 제거, 절단 처리 global hook 설치 또는 제거
`rename-tab`, `set-status`, OSC title sequence로 가볍게 시작 `UserPromptSubmit`에서 validator, `codex exec`, broad repo scan 실행 socket auth, global cmux/Warp/Oz/shell settings 수정
