cmux-automation

cmux 탭, 상태, markdown board, Codex hook을 연결해 현재 탭에는 짧은 작업 라벨을, status에는 원문 맥락을 남기는 자동화 스킬.

에이전트 지시문SKILL.md 사용자 가이드skill.html 상세 레시피hook-recipes.md

사용 판단 매트릭스

사용

질문 고정

Codex에 제출한 질문을 짧은 작업 라벨과 전체 status에 자동 반영하고 싶을 때.

함께 사용

세션 보드

여러 agent/pane의 질문, 성공 조건, 상태를 markdown panel로 유지할 때.

확인

전역 설정

global cmux 설정, hook 설치/삭제, socket auth 변경은 사용자 확인을 먼저 받는다.

건너뜀

파괴 조작

pane close, history clear, send-key 같은 조작은 명시 요청 없이는 하지 않는다.

자동화 흐름

입출력 스키마

입력

  • cmux 작업 요청
  • Codex hook payload
  • workspace/tab/pane 목표

처리

  • UserPromptSubmit은 짧게 유지
  • CMUX_SURFACE_ID로 현재 surface 우선 지정
  • tab에는 규칙 기반 작업 라벨 표시
  • 원문은 current-question status에 보존

출력

  • 짧은 작업 라벨
  • 전체 질문 status
  • hook script 또는 config

금지와 허용

허용
  • 작업 라벨을 tab title에 고정한다.
  • 질문 원문은 status나 local markdown으로 유지한다.
  • cmux 밖에서는 조용히 no-op 처리한다.
금지
  • 요청 없이 global hook을 설치하지 않는다.
  • socket password나 secret을 파일에 저장하지 않는다.
  • pane close, clear-history, send-key를 임의 실행하지 않는다.
