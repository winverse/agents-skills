cmux-automation

cmux 탭, status, markdown board, Codex hook을 연결해 현재 작업 질문을 짧게 보이게 하고 원문 맥락은 잃지 않게 유지하는 자동화 스킬.

사용 판단 매트릭스

사용

질문 고정(prompt pinning)

Codex에 제출한 질문을 cmux tab title과 `current-question` status에 자동 반영해야 할 때.

함께 사용

세션 작업판

여러 agent, pane, workspace의 질문과 완료 조건을 `cmux markdown open`으로 추적할 때.

먼저 확인

전역 변경

global hook, global cmux settings, socket auth, 다른 pane 입력은 사용자 승인이 필요하다.

건너뜀

파괴적 UI 조작

pane 닫기, workspace 이동, history 삭제, send-key 실행은 명시 요청 없이는 하지 않는다.

자동화 흐름

기본 경로 조용한 fallback

입출력 스키마

입력

  • cmux 자동화 요청
  • Codex `UserPromptSubmit` JSON payload
  • `CMUX_WORKSPACE_ID`, `CMUX_SURFACE_ID`, `CMUX_TAB_ID`

처리 규칙

  • `오케이`, `okay`, `ㅇㅇ`, `그리고`, `이제`, `좋아` 같은 filler 제거
  • 반복되는 작업은 빠른 fallback 라벨로 축약
  • `CMUX_SURFACE_ID`를 먼저 쓰고 `cmux identify`는 fallback으로만 사용

출력

  • `cmux rename-tab --surface ...` 또는 `--tab ...`
  • `cmux set-status current-question ... --icon pin`
  • 승인이 필요한 Codex `/hooks` 안내

의미 요약 분리

hook 단계

빠른 fallback 라벨

`UserPromptSubmit`은 agent가 prompt 의미를 이해하기 전에 실행된다. 그래서 hook은 진짜 요약이 아니라 빠른 규칙 라벨과 원문 status만 만든다.

agent 단계

의미 기반 tab title

agent가 요청을 읽은 뒤 `cmux rename-tab`으로 `hook 요약 구조 검토`처럼 더 정확한 짧은 제목을 덮어쓴다.

우선순위 차트

기본 구현은 current surface를 빠르게 갱신한다. Codex hook 실행 surface와 agent command surface가 갈라지는 경우에만 `CMUX_PIN_PROMPT_SCOPE=workspace`를 사용한다.

`project-snippets/cmux-automation.md`는 프로젝트별 `AGENTS.md`, `CLAUDE.md`, 또는 동등한 instruction 파일에서 이 스킬을 연결할 때 쓰는 조각이다.

품질 게이트

node scripts/validate-skill.ts skills/cmux-automation
node skills/cmux-automation/scripts/validate-cmux-automation.ts skills/cmux-automation
node scripts/validate-skill-html.ts .
node scripts/validate-skill-repo.ts .
printf '{"prompt":"오케이 그리고 workflow 스킬을 만들어줘"}' \
  | CMUX_PIN_PROMPT_DRY_RUN=1 \
    node skills/cmux-automation/scripts/cmux-pin-prompt.mjs

실제 hook 구현에서는 dry-run으로 title 규칙을 먼저 확인하고, cmux 안에서 tab/status 변경과 지연 시간을 별도로 확인한다.

금지와 허용

허용
  • project-local hook script와 hook config를 백업 후 갱신한다.
  • tab에는 빠른 fallback 라벨, status에는 원문 질문을 보존한다.
  • agent가 의미를 이해한 뒤 더 나은 제목으로 tab을 덮어쓴다.
  • cmux가 없거나 cmux 밖이면 ergonomic hook은 조용히 no-op 처리한다.
금지
  • 요청 없이 global hook 또는 global cmux settings를 바꾸지 않는다.
  • socket password, token, secret을 hook script나 로그에 저장하지 않는다.
  • `UserPromptSubmit` 안에서 validator, `codex exec`, broad repo scan을 실행하지 않는다.
