Transcript Polisher

전사본을 코드 치환이 아니라 에이전트가 직접 읽고, source/output 구조와 output mode를 지키며 Claude Code `/goal`식 완료 조건 루프로 평가자가 납득할 증거가 나올 때까지 다듬는 로컬 스킬.

파일 쌍
SKILL.md 에이전트용 규칙
skill.html 사용자용 안내

사용 판단: 직접 읽어서 고칠 때만 사용

사용

STT 전사본, 강의 대본, 자막, 회의록의 문맥과 흐름을 다듬을 때.

함께 사용

오디오 변환 뒤 source root와 output root를 보존하면서 Claude Code `/goal`처럼 완료 조건까지 반복 검증할 때.

건너뜀

단순 파일명 변경, 포맷 변환, 명확한 한두 글자 오타 수정만 필요할 때.

중단/확인

오디오 확인 없이는 확정할 수 없는 고유명사나 숫자가 핵심 의미일 때.

실행 흐름: 분할, 병렬 검토, 목표 감사

1

루트 결정

source root, output root, output mode를 먼저 고정한다.

2

전체 읽기

파일과 강의 흐름을 먼저 파악한다.

3

용어 장부

확정 용어와 불확실 용어를 나눠 둔다.

4

단위 분할

문단, 구문, 주제 전환 기준으로 나눈다.

5

통합 편집

메인 에이전트가 말투와 용어를 맞춘다.

6

목표 감사

구조, 분량, 증거가 부족하면 다음 루프로 돌아간다.

Claude Code `/goal` 접목: 완료 조건이 충족될 때까지 반복

완료 조건source/output root, output mode, 품질 기준, 최대 턴 조건을 한 문장으로 적는다.
증명 방법상대 경로, 파일 수, 분량 보존, 검증 결과처럼 평가자가 볼 증거를 정한다.
대화에 노출Claude Code 평가자는 도구를 쓰지 않으므로 증거를 보고에 남긴다.
개선 한계자가 개선 스킬이 아니라, 드러난 증거로 완료 여부를 감사하는 루프다.
계속/종료조건이 대화상 증거로 충족되지 않으면 다음 턴으로 이어간다.

입력과 출력: 원문 의미는 보존

입력

  • 전사본 .txt 파일
  • source root 또는 파일 목록
  • output root
  • polish, lecture-notes, summary mode
  • 원본 수정 또는 병렬 출력 지시

처리

  • 직접 읽기
  • 문맥 단위 분할
  • 확정/불확실 용어 장부
  • 서브 에이전트별 섹션 검토
  • 메인 에이전트 최종 통합
  • Claude Code `/goal`식 완료 조건 루프

출력

  • 같은 상대 경로의 수정 전사본
  • 불확실한 표현 목록
  • source/output 파일 수와 분량 검증
  • 검증 결과
  • 완료 조건 증거 목록
  • 파일/라인 기준 요약

파일 관계도: 공유 스킬 패키지 구성

skills/transcript-polisher/

└── agents/
    └── openai.yaml

연결 방식

이 스킬은 공유 skills repo의 source instruction으로 둔다. 각 프로젝트에서는 필요한 경우 instruction 파일에서 이 경로를 명시해 연결한다.

사용자가 요청하지 않으면 전역 ~/.codex/skills에 설치하지 않고 프로젝트별 연결을 우선한다.

검증 게이트: 끝내기 전에 다시 읽기

조건Claude Code 평가자가 볼 수 있는 완료 조건인가.
범위source/output 파일 수와 상대 경로가 일치하는가.
보존source root는 원본이고 수정은 output root에만 있는가.
분량polish가 요약본처럼 과도하게 줄지 않았는가.
증거불확실 용어와 검증 증거를 보고에 남겼는가.

금지와 허용: 코드 치환으로 돌아가지 않기

해야 함

  • 문단 단위로 직접 읽고 판단한다.
  • source/output root와 output mode를 먼저 고정한다.
  • Claude Code `/goal`처럼 완료 조건과 평가자용 증거를 유지한다.
  • 허용된 경우에만 서브 에이전트 결과를 메인에서 다시 통합한다.
  • 확정 용어와 불확실 용어를 분리한다.
  • 불확실한 고유명사는 확인 대상으로 남긴다.

하지 말 것

  • 치환표나 regex로 본문을 자동 수정하지 않는다.
  • 평가자가 볼 증거 없이 완료라고 말하지 않는다.
  • 서브 에이전트를 쓰지 않았는데 썼다고 말하지 않는다.
  • source root에 검토본 패치를 잘못 적용하지 않는다.
  • 검토본을 단순 복사한 채 완료라고 말하지 않는다.
  • 원문에 없는 설명이나 사실을 추가하지 않는다.
  • 강의 예시의 의도적 어색함을 임의로 매끈하게 만들지 않는다.
