code-review

Findings-first 코드 리뷰 스킬. 버그, 회귀, 누락 테스트, SRP/SOLID 경계, JS/TS collection style을 우선 검토한다.

에이전트SKILL.md 사용자skill.html

사용 판단 매트릭스

사용

리뷰 요청

PR, diff, implementation audit, regression risk, missing tests 검토.

초점

발견사항 우선

요약보다 버그, 보안, 데이터 손실, 회귀 위험을 먼저 둔다.

함께 사용

브라우저/디자인

렌더 증거는 browser-qa, 시각 취향 판단은 design-review와 결합한다.

건너뜀

수정 요청

사용자가 fix를 요청하지 않았다면 리뷰 중 코드를 고치지 않는다.

리뷰 흐름

입출력 스키마

입력

  • diff/PR/파일 경로
  • 요구사항과 테스트 결과
  • JS/TS style context

리뷰 축

  • behavior/security
  • SRP/SOLID
  • map/filter/reduce/Iterable

출력

  • severity findings
  • file:line 근거
  • tests gap

금지와 허용

허용
  • Findings를 먼저 쓴다.
  • file:line과 영향도를 붙인다.
  • missing tests를 실제 회귀 위험과 연결한다.
금지
  • 칭찬과 요약으로 findings를 묻지 않는다.
  • 루프가 더 명확한 곳에 functional style을 강요하지 않는다.
  • 요청 없이 코드를 수정하지 않는다.
