Atomic Committer

커밋 요청을 받으면 dirty tree를 먼저 읽고, 비밀값 검사와 forbidden-content scan을 거친 뒤 작업 단위로 stage, 재검사, commit, 조건부 push까지 수행하는 스킬입니다. 메시지는 영어 prefix + 한글 메시지이며 remote 있을 때만 push합니다.

파일 쌍 SKILL.md 에이전트 지시문 skill.html 사람용 시각 가이드

사용 판단

커밋 요청이 있어도 자동으로 전체 stage하지 않습니다. 먼저 intent, diff, staged 상태, secret 위험을 분리합니다.

사용

사용자가 commit, commit and push, 여러 변경을 atomic commit으로 나누라고 요청했습니다.

분리 또는 결합

기능, 테스트, 문서, 생성물이 같은 의도에 묶이면 하나의 changeset으로 다루고, 무관한 변경은 분리합니다.

질문

한 파일에 unrelated hunk가 섞였거나 staged 변경이 이미 있고 사용자 의도와 충돌할 수 있습니다.

중단

live-looking credential assignment, private key, 실제 인증 URL은 사용자 요청으로도 커밋할 수 없습니다.

실행 흐름

검사는 두 번 합니다. candidate diff를 보고 그룹을 정한 뒤 staged diff를 다시 검사해야 commit할 수 있습니다.

일반 단계 중간 gate 화살표: 다음 행동

우선순위와 위험

그룹 기준은 파일 종류가 아니라 의도입니다. 다만 secret 안전과 `.gitignore` 예방은 커밋보다 앞섭니다.

강제 차단실제값처럼 보이는 cloud, source control, AI API, payment, messaging, package registry token assignment. 예: AWS_ACCESS_KEY=..., OpenAI API key
DB URLDATABASE_URL, REDIS_URL, MONGODB_URI, POSTGRES_URL 안의 실제 username/password 또는 token
키 재료AKIA..., ASIA..., JWT-shaped value, private key block
경고 후 확인.env.example, placeholder docs, credential-like path, local screenshot, 큰 생성물
placeholderexample, dummy, REPLACE_ME, ${...}, sk-...REDACTED는 주변 diff를 보고 판단합니다.

입출력 스키마

이 스킬은 git 작업을 수행하지만 최종 보고도 계약입니다. 무엇을 커밋했고, 무엇을 남겼고, 왜 push했는지 또는 생략했는지 알려야 합니다.

입력

요청commit, commit and push, split commits
상태git status --short, working diff, staged diff, untracked files
제약사용자 변경 보존, destructive command 금지, force-commit 금지

처리

검사candidate diff와 staged diff의 forbidden-content triage
정리반복 local artifact는 좁은 `.gitignore` 패턴으로 예방
검증관련 validation이 practical하면 실행하고 실패 시 push 전 중단

출력

commitshash와 메시지
skipped남은 dirty file, 차단 이유, 사용자 확인 필요 항목
pushremote/branch 또는 생략 사유

`.gitignore` 예방 gate

`.gitignore`는 이미 staged된 비밀값을 숨기는 우회로가 아니라, untracked local artifact가 반복 유입되지 않게 막는 예방 장치입니다.

오용 방지

atomic commit 자동화가 사용자 변경 손실, secret 유출, 예기치 않은 push로 이어지지 않게 막는 규칙입니다.

허용논리 changeset별로 파일이나 hunk를 선택 stage합니다.
허용반복 untracked local artifact는 좁은 `.gitignore` 패턴으로 예방합니다.
허용commit message는 <type>: <한글 요약> 형식으로 씁니다.
허용push 요청, remote, branch, validation이 모두 맞을 때만 publish합니다.
금지git add .로 무관한 변경을 한 번에 포함하지 않습니다.
금지hard-blocked credential이나 private key를 force request로 커밋하지 않습니다.
금지tracked secret을 `.gitignore` 변경으로 숨기려 하지 않습니다.
금지reset --hard, checkout --, amend, rebase, force-push는 명시 요청 없이는 하지 않습니다.

## Project Skills

- Use $atomic-committer at <skills-root>/skills/atomic-committer/SKILL.md when asked to commit, split changes into commits, create multiple logical commits, or commit and push.

## Project Skill Overrides

- Scan candidate changes before staging and scan `git diff --cached -U0 --no-ext-diff` before committing.
- Hard-block live-looking credential assignments, private-key material, and URL credentials across common providers.
- Update `.gitignore` only as narrow prevention for untracked local or secret-bearing artifacts; never use it to hide tracked secret content.
- Group files by logical changeset and use commit messages like `feat: 한글 요약`.
- Push only when the user requested push, a remote exists, the branch is not detached, and validation is not blocking.
