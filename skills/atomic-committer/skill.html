Atomic Committer

dirty git tree를 Secret Guard로 먼저 검사하고 작업 단위로 나눠 커밋하며, 메시지는 영어 prefix + 한글 메시지로 작성하고 remote 있을 때만 push합니다.

AC

사용 판단 매트릭스

커밋 요청이 있어도 무조건 `git add .`로 가지 않습니다.

바로 사용

사용자가 commit, commit and push, 변경사항을 나눠 커밋하라고 요청했습니다.

먼저 분리

dirty file이 여러 관심사를 담고 있으면 diff를 읽고 atomic commit 단위로 분류합니다.

확인 필요

한 파일 안에 unrelated hunk가 섞였거나 placeholder secret 예시, 큰 생성물, local artifact가 있습니다.

강제 차단

실제값처럼 보이는 credential assignment, private key, live token은 강제 요청이 있어도 커밋하지 않습니다.

비밀값 검사

candidate diff와 staged diff를 둘 다 검사합니다.

강제 차단 (Hard block)AWS, GCP, Azure, GitHub, OpenAI, Stripe, Slack, npm, DB URL / DB URLs의 실제값처럼 보이는 인증값 할당
AWS 예시AWS_ACCESS_KEY=... 값이 실제처럼 보일 때만 차단하고, placeholder 언급은 경고로 둡니다
키 패턴AKIA[0-9A-Z]{16}, ASIA[0-9A-Z]{16}, JWT-shaped values, private key block
먼저 경고.env.example, placeholder docs, credential-like paths, local screenshots, generated files
우회 금지--no-verify, force request, manual staging으로 forbidden-content scan을 우회하지 않습니다.

커밋 메시지 규칙

접두사는 conventional commit, 본문은 한글로 고정합니다.

형식<type>: <한글 요약>
범위(scope) 선택<type>(<scope>): <한글 요약>
예시feat: 변경사항 단위 커밋 스킬 추가
주요 typefeat, fix, docs, refactor, test, chore, build, ci, style, perf, revert

금지와 허용

커밋 자동화가 위험한 동작으로 흐르지 않게 막는 misuse guardrails입니다.

허용diff를 읽고 logical changeset별로 stage합니다.
허용candidate diff와 staged diff를 모두 Secret Guard로 검사합니다.
허용커밋 메시지는 영어 prefix와 한글 요약으로 씁니다.
금지`git add .`로 무관한 변경을 한 번에 포함하지 않습니다.
금지live-looking credential, private key, local artifact를 커밋하지 않습니다.
금지remote가 없거나 push 요청이 없으면 push하지 않습니다.

## Project Skills

- Use $atomic-committer at <skills-root>/skills/atomic-committer/SKILL.md when asked to commit, split changes into commits, create multiple logical commits, or commit and push.

## Project Skill Overrides

- Commit messages must use an English conventional prefix such as `feat`, `fix`, `docs`, or `chore`, followed by a Korean summary.
- Group changed files by logical changeset before committing.
- Before staging and before committing, scan candidate changes for forbidden content. Hard-block live-looking credential assignments and private-key material across common providers. For example, block `AWS_ACCESS_KEY=...` only when the assigned value looks real rather than placeholder text, and do not allow force-commit overrides.
- Push only when a git remote exists and the user requested push.
