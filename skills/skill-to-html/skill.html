skill-to-html

모든 스킬 설명을 같은 문법으로 정리하고, 사용 판단과 실행 흐름을 빠르게 읽히는 `skill.html`로 만든다.

필수 파일 쌍 skills/<skill-name>/SKILL.md skills/<skill-name>/skill.html

사용 판단 매트릭스

스킬 변경 상태로 use, revise, skip, ask를 결정한다.

바로 사용

`SKILL.md`가 생성 또는 큰 폭으로 수정됐고 `skill.html`이 없거나 오래된 경우.

수정

기존 HTML이 카드형 설명에 가깝거나, diagram-rich 기준과 맞지 않는 경우.

건너뜀

스킬 내용이 바뀌지 않았고 HTML, validator, snippet이 모두 최신인 경우.

확인 후 결합

스킬 목적이 불명확하거나 `skill-creator`, `skill-update` 판단이 먼저 필요한 경우.

skill.html 생성 흐름

기존 HTML을 보존할지 판단한 뒤, 설명 문법과 검증을 맞춘다.

수정 전 기존 `skill.html`을 읽고 유지할 도표와 버릴 구조를 먼저 분리한다.
수정 중 스킬마다 같은 설명 구조를 쓰되, 도표 내용은 해당 스킬의 실제 workflow에 맞춘다.
완료 정적 validator와 PC viewport 렌더링 확인을 통과해야 완료로 본다.

파일 관계 지도

사람은 HTML을 먼저 보고, agent는 `SKILL.md`와 references를 읽는다.

SKILL.md에이전트 흐름 skill.html사용자용 시각 가이드 시각 기준디자인 기준 기본 검증기frontmatter 검사 스킬 검증기HTML 가이드 검사 project-snippets/skill-to-html.md프로젝트 연결
로컬 파일 상대 링크만 사용한다. 외부 CDN, 이미지, script, font 의존성은 만들지 않는다.
공유 문서 trigger, workflow, validator, snippet, lifecycle가 바뀌면 repo 문서도 같이 맞춘다.
검사관 검사 이슈는 local-only `inspector/`에 적고, 해결되면 삭제한다.

품질 게이트

스킬 HTML은 예뻐 보이는 것보다 반복 검증 가능한 상태가 중요하다.

기본 스킬 검증node scripts/validate-skill.ts skills/<skill-name>
스킬별 검증스킬별 `scripts/validate-*.ts`가 있으면 함께 실행한다.
PC 렌더링PC viewport에서 console error, overlap, page overflow, broken local link를 확인한다.
repo 동기화필요하면 `README.md`, `AGENTS.md`, snippets, history까지 맞춘다.

금지와 허용

카드 나열형 문서를 피하고, 읽는 목적에 맞는 도표를 사용한다.

허용

  • 목적, 사용 판단, workflow, output, resource, validation, misuse 순서를 유지한다.
  • diagram에는 직접 라벨과 짧은 legend를 붙인다.
  • 긴 경로와 명령은 줄바꿈 또는 code block overflow로 처리한다.
  • 기존 HTML의 좋은 구조는 보존하고 stale한 부분만 고친다.

금지

  • 스킬 설명을 같은 크기의 카드 묶음으로만 나누지 않는다.
  • 일반 flow나 matrix를 검은 배경 장식 블록으로 만들지 않는다.
  • 색만으로 상태를 구분하지 않는다.
  • 외부 CDN, web font, image, script에 의존하지 않는다.

프로젝트 연결

프로젝트 instruction에는 스킬 링크와 최소 검증 규칙만 둔다.

## Project Skills

- Use $skill-to-html at <skills-root>/skills/skill-to-html/SKILL.md whenever a skill is created, installed, forked, or updated, so the skill folder gets a diagram-rich skill.html beside SKILL.md.

## Validation

- Run node scripts/validate-skill.ts skills/<skill-name> before calling the skill done.
- Run the skill-specific TypeScript validator when one exists.
- Verify skill.html in a PC desktop viewport when practical.
