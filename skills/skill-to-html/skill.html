skill-to-html

SKILL.md를 도표 중심 skill.html로 바꾸고 repo 검증 흐름까지 맞추는 스킬

Required pairSKILL.md + skill.html

사용 판단 매트릭스

스킬 폴더의 상태로 use, skip, combine, ask를 결정합니다.

Use now

SKILL.md가 생성/수정됐고 skill.html이 없거나 오래된 경우 바로 사용합니다.

Revise

skill.html은 있지만 카드형 요약이거나 diagram-rich 기준에 못 미치면 고칩니다.

Skip

스킬 변경이 없고 기존 skill.html이 최신이며 검증도 통과한 경우 건너뜁니다.

Ask / combine

스킬 목적이 불명확하거나 skill-creator 구조 작업이 먼저 필요하면 확인하거나 함께 씁니다.

판독 목적별 시각 패턴

글 덩어리를 나누는 대신 읽는 목적에 맞는 도표를 선택합니다.

언제 쓰나?

Decision matrix로 use, skip, combine, ask를 구분합니다.

무엇을 먼저 하나?

Flowchart나 timeline으로 agent의 실행 순서를 보여줍니다.

무엇이 중요한가?

Bar chart, funnel, meter로 우선순위와 위험도를 표현합니다.

어떤 파일이 있나?

Resource map으로 SKILL.md, references, scripts, assets를 연결합니다.

skill.html 생성 흐름

스킬 변경 뒤 HTML, TS 검증, history 정리를 함께 처리합니다.

필수 시각 요소 비중

새 skill.html은 최소 네 종류의 시각 구조를 포함합니다.

금지와 허용

현재 HTML처럼 단순 파티션만 나누는 형태를 피합니다.

Do

의사결정, 순서, 우선순위, 파일 구조를 도표로 보여줍니다.

Do not

카드 몇 개에 설명문을 나눠 넣는 방식으로 끝내지 않습니다.

Do

색상 외에도 위치, 라벨, 선, 형태로 의미를 전달합니다.

Do not

장식용 그래픽, 3D 차트, 과한 그림, 읽기 어려운 범례를 쓰지 않습니다.

프로젝트 연결

스킬 생성 프로젝트에는 skill-creator, TS validator와 함께 연결합니다.

## Project Skills - Use $skill-to-html at <skills-root>/skills/skill-to-html/SKILL.md whenever a skill is created, installed, forked, or updated, so the skill folder gets a diagram-rich skill.html beside SKILL.md. ## Validation - Run node scripts/validate-skill.ts skills/<skill-name> before calling the skill done. - Use repo-owned .ts validators, not Python validators.
