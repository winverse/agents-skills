skill-to-html

`SKILL.md` 옆에 사람이 빠르게 이해할 수 있는 diagram-rich `skill.html`을 만든다. 설명 문단보다 decision matrix, workflow, resource map, input/output schema, risk chart처럼 판단 가능한 구조를 우선한다.

호출 조건 스킬이 created, installed, forked, updated 되었거나 adjacent visual guide가 stale 또는 missing인 경우.
오용 방지 `SKILL.md`, references, docs, snippets, validators를 고치지 말라는 요청이 있으면 target `skill.html`만 갱신한다.

사용 판단 매트릭스

언제 만들고, 언제 멈추고, 무엇을 확인할지 먼저 결정한다.

바로 생성

`skill.html`이 없고 `SKILL.md`가 현재 source of truth인 경우. 새 HTML은 self-contained static file이어야 한다.

갱신

HTML이 오래됐거나 카드 요약에 머물러 trigger, workflow, guardrail, output shape가 보이지 않는 경우.

건너뜀

source instruction이 바뀌지 않았고 diagram, validation command, PC desktop layout이 최신인 경우.

범위 차단

hook이나 사용자 요청이 adjacent `skill.html`만 허용하면 다른 파일 동기화는 하지 않는다.

생성 흐름

SVG arrow endpoint가 실제 node에 닿도록 배치하고, 넓은 표는 full-width section에 둔다.

1. 읽기대상 `SKILL.md`와 필요한 `references/`만 읽고 trigger와 guardrail을 확정한다.
2. 설계decision matrix, workflow, resource map, input/output schema, risk/guardrail chart를 배치한다.
3. 검증arrow endpoint, table width, overflow, text overlap을 확인하고 `node scripts/validate-skill-html.ts .`를 실행한다.

파일 관계 지도

사람은 `skill.html`을 읽고, agent는 `SKILL.md`와 필요한 references를 읽는다.

프로젝트 연결 snippet

프로젝트 instruction에는 스킬 링크와 최소 완료 조건만 둔다.

- Use $skill-to-html at skills/skill-to-html/SKILL.md whenever a skill is created, installed, forked, or updated, so the skill folder gets a diagram-rich skill.html beside SKILL.md.
- Keep output static HTML with embedded CSS/SVG and no external assets.
- Validate with node scripts/validate-skill-html.ts . and inspect arrow endpoint, table width, overflow, text overlap in PC desktop viewport when the visual structure materially changes.
- Ignore mobile/tablet layout, responsive breakpoint tuning, and touch interaction for this skill.
