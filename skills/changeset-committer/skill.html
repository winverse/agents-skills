Changeset Committer

dirty git tree를 작업 단위로 나눠 커밋하고, 메시지는 영어 prefix + 한글 메시지로 작성하며, remote 있을 때만 push합니다.

CC

사용 판단 매트릭스

커밋 요청이 있어도 무조건 `git add .`로 가지 않습니다.

Use now

사용자가 commit, commit and push, 변경사항을 나눠 커밋하라고 요청했습니다.

Split first

dirty file이 여러 관심사를 담고 있으면 diff를 읽고 changeset으로 분류합니다.

Ask

한 파일 안에 unrelated hunk가 섞여 있고 안전하게 나누기 어렵습니다.

Skip push

remote가 없거나 push 요청이 없거나 detached HEAD 상태입니다.

커밋 메시지 규칙

접두사는 conventional commit, 본문은 한글로 고정합니다.

형식<type>: <한글 요약>
scope optional<type>(<scope>): <한글 요약>

## Project Skills

- Use $changeset-committer at <skills-root>/skills/changeset-committer/SKILL.md when asked to commit, split changes into commits, create multiple logical commits, or commit and push.

## Project Skill Overrides

- Commit messages must use an English conventional prefix such as `feat`, `fix`, `docs`, or `chore`, followed by a Korean summary.
- Group changed files by logical changeset before committing.
- Push only when a git remote exists and the user requested push.
