스킬 시각 가이드

show-skills

현재 파일 시스템을 source of truth로 읽어 스킬 목록을 카테고리별로 보여주고, 작업에 맞는 주 스킬과 companion skill을 추천하는 catalog 탐색 스킬.

에이전트SKILL.md 사용자skill.html 실행scripts/show-skills.ts

사용 판단 매트릭스

사용

목록과 경로

현재 repo 또는 skills root의 스킬 이름, 설명, SKILL.md, skill.html 경로를 보여줄 때.

조합

무엇을 쓸지 선택

사용자 작업을 보고 primary skill과 함께 쓰면 좋은 companion skill을 짧게 추천한다.

확인

다른 카탈로그 root

현재 위치가 catalog가 아니면 --root <path> 또는 SKILLS_ROOT로 대상 root를 지정한다.

건너뜀

실행이나 설치

스킬을 자동 설치, global 등록, 실제 작업 수행으로 이어가지 않는다. 목록과 추천까지만 한다.

목록 생성 흐름

catalog 탐색 경로 추천 판단 보강

입출력 스키마

입력

  • 현재 스킬 보여줘
  • 이 작업에는 어떤 스킬을 써?
  • --root skills, --compact, --format json
→

출력

  • 카테고리별 스킬 목록
  • 상태/risk와 간단 설명
  • SKILL.md / skill.html 링크
  • 주 스킬 / companion 추천

소스 우선순위 차트

현재 파일 시스템항상 우선

핵심 판단

정적 문서가 있더라도 live listing(실시간 목록)을 먼저 신뢰한다. docs/skill-catalog.md는 사람이 훑는 보조 카탈로그이며, standalone install에는 없을 수 있다.

금지와 허용

허용

  • 현재 파일 시스템에서 스킬을 읽고 stale 수동 목록을 보조 자료로만 본다.
  • repo layout과 installed layout 모두 지원한다.
  • 간단 요청에는 compact output을 쓰고, 자동화에는 JSON output을 쓴다.
  • 추천이 필요하면 primary skill, companion skill, 이유, exact paths를 함께 준다.

금지

  • 목록 요청 중 스킬을 install, symlink, global register하지 않는다.
  • 사용자가 실행을 요청하지 않았는데 추천한 스킬을 바로 수행하지 않는다.
  • downloaded 또는 standalone catalog에 docs/skill-catalog.md가 반드시 있다고 가정하지 않는다.
  • 경로를 보여줄 때 local absolute path를 고정하지 않는다.
