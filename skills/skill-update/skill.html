skill-update

기존 shared skill을 바꿀 때 호출 즉시 original/upstream provenance preflight를 실행하고, source delta를 adopt/adapt/reject/defer로 분류해 repo-local package와 검증 증거를 한 흐름으로 묶는 유지보수 스킬입니다.

skills/skill-update/ SKILL.md - agent workflow skill.html - human visual guide

사용 판단 매트릭스

기존 스킬 변경 요청인지, 원본 확인이 필요한 fork/update인지, 먼저 물어야 하는 변경인지 구분합니다.

사용

기존 shared skill의 trigger, workflow, reference, validator, snippet, docs, history, lifecycle를 수정할 때.

원본 확인

모든 호출에서 local provenance를 먼저 훑고, 원본 skill, upstream primitive, fork 기록이 있거나 의심되면 `web-research`로 공식 source를 검증합니다.

질문 먼저

trigger 의미 변경, behavior 제거, rename, split, archive, repo-wide preference 변경, source-of-truth conflict.

다른 스킬

새 스킬 scaffold는 system `skill-creator`, visual guide는 `skill-to-html`, 문서 drift는 `sync-docs`. plugin package, MCP config, submodule update는 `Plugin update list` 확인 후 별도 plugin workflow입니다.

업데이트 흐름

사용자 의도에서 시작해 원본 후보를 자동으로 찾고, source ledger와 adopt/adapt/reject/defer 판단을 남긴 뒤 최소 파일 수정과 검증까지 닫습니다.

입출력 스키마

요청은 behavior change이고, 결과는 검증된 shared skill package입니다.

입력

  • 대상 skill folder
  • 원하는 행동 변화 또는 실패 증거
  • 현재 `SKILL.md`, references, snippets, history
  • 원본/fork/upstream 가능성
→

출력

  • 최소 수정된 skill package
  • 최신 `skill.html`
  • source ledger와 checked date
  • adopt/adapt/reject/defer 판단
  • validator 결과와 남은 질문

원본 확인 의사결정

원본 여부를 사용자가 말하지 않아도 먼저 provenance preflight를 실행하고, 원본 delta가 있으면 local repo에 맞는 항목만 반영합니다.

파일 관계 지도

`SKILL.md`는 짧게, 긴 기준은 references와 validator에 두고, material change는 HTML과 history까지 닫습니다.

검증과 완료 보고

작업 범위가 커질수록 repo 검증 명령을 빠뜨리지 않고, 실패하면 남은 risk를 명시합니다.

node scripts/validate-skill.ts skills/<target-skill>
node skills/<target-skill>/scripts/<skill-validator>.ts
node scripts/validate-korean-markdown.ts .
node scripts/validate-skill-html.ts .
node scripts/run-agent-evals.ts

# add/remove/rename/archive/restore가 있으면:
node skills/show-skills/scripts/update-html-catalog.ts skills/show-skills

해야 할 일

  • 사용자가 의도한 behavior change에서 시작합니다.
  • 모든 호출에서 original/upstream provenance preflight를 실행합니다.
  • 원본이 있거나 의심되면 provenance와 공식 source를 확인합니다.
  • plugin-bundled skill은 metadata를 원본 증거로만 읽습니다.
  • `SKILL.md`는 concise하고 trigger-focused로 유지합니다.
  • material change면 `skill-to-html`을 실행하고 검증 결과를 보고합니다.

멈춰야 할 일

  • 요청 없이 기존 유용한 behavior를 제거하지 않습니다.
  • 원본 전체를 이 repo에 무조건 복사하지 않습니다.
  • plugin package, MCP config, submodule 자체는 `Plugin update list` 확인 후 별도 workflow로 넘깁니다.
  • source-of-truth conflict를 추측으로 해결하지 않습니다.
  • rename, split, archive, repo-wide preference 변경을 임의로 진행하지 않습니다.
  • HTML, snippets, docs, history를 stale 상태로 남기지 않습니다.
