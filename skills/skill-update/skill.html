Decision Matrix

기존 스킬 수정 요청인지, 다른 스킬을 먼저 써야 하는지 빠르게 구분합니다.

Use

기존 스킬의 trigger, workflow, references, validator, snippet, lifecycle를 바꿀 때.

Combine
  • material change → skill-to-html
  • 문서 drift 가능 → sync-docs
  • 커밋 요청 → atomic-committer
Ask

트리거 변경, rename, split, merge, deprecate, 제거처럼 의도가 불명확하면 수정 전에 확인합니다.

Skip

새 스킬의 초기 scaffold만 필요하면 먼저 system skill-creator를 사용합니다.

Input / Output

Input

  • 업데이트할 스킬 이름
  • 원하는 행동 변화
  • 현재 파일과 history
  • 검증 실패 또는 사용자 피드백
→

Output

  • 수정된 skill package
  • 최신 skill.html
  • 맞춰진 snippets/docs/history
  • validator 결과와 남은 질문

Project Connection

스킬을 운영하는 프로젝트에만 붙입니다.

## Project Skills

- Use $skill-update at <skills-root>/skills/skill-update/SKILL.md when asked to update, revise, improve, maintain, rename, split, deprecate, or otherwise change an existing shared skill.

## Project-Specific Overrides

- Keep SKILL.md, references, validator, agents/openai.yaml, skill.html, snippets, docs, and history aligned when behavior changes.
- Use skill-to-html after material skill changes.
- Use sync-docs when paths, snippets, README, AGENTS, docs, history, or validation commands may drift.
- Ask before changing unclear trigger behavior, lifecycle state, rename, split, merge, deprecation, or removal.

Do

  • 사용자가 원하는 행동 변화를 먼저 확인합니다.
  • 한 파일만 고치지 말고 관련 package 전체를 맞춥니다.
  • validator와 history 기준을 끝에서 확인합니다.

Do Not

  • 애매한 trigger 변경을 임의로 결정하지 않습니다.
  • SKILL.md에 긴 취향과 예시를 계속 쌓지 않습니다.
  • skill.html, snippets, README를 stale 상태로 남기지 않습니다.
